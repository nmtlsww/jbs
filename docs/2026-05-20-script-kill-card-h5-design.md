# 剧本杀体验卡 H5 — 设计文档

- 日期：2026-05-20
- 状态：待评审
- 设计源：`reference/original-design.png`（941 × 1672，RGB）

---

## 1. 背景与目标

把一张已压平的设计稿（剧本杀体验卡活动页）做成一个 **可交互的纯静态 H5**：

- 视觉 100% 还原设计稿
- 点击"使用"按钮：剩余次数 −1，使用记录新增一条
- 无后端、无接口，所有状态存在浏览器本地

## 2. 范围 / 非目标

**范围内：**
- 单页 H5，移动端为主
- 1 个可点击按钮 + 2 个动态区域（剩余次数数字、使用记录列表）

**非目标（明确不做）：**
- 不接任何后端接口、不做账号体系
- 不做卡片类型切换（固定为 30 次卡）
- 不做重置按钮、分享、埋点等任何未被要求的功能
- 不为"未来可能"预留抽象或配置开关

## 3. 技术栈与约束

- 原生 `HTML + CSS + JavaScript`，**零依赖、零构建、无框架**
- 不可直接用整图当 `background-image` + `cover`（这是其他实现错位的根因）
- 美术部分（人物/背景/烫金边框/标题/卡片/按钮/静态文字）**一律用图片**，前端不写代码还原
- 仅 3 个元素是代码：剩余次数数字、使用记录列表、按钮热区

## 5. 页面结构与响应式策略

```
<body>                       背景色 #0A0A12（宽屏留白处）
  <div class="stage">        舞台：限宽居中，宽高比锁定 941/1672
    <img class="bg">         干净底图，width:100%; display:block —— 由它定义舞台尺寸
    <div class="overlay">    交互层：position:absolute; inset:0; container-type:inline-size
      <div class="count">       剩余次数数字
      <div class="records">     使用记录滚动容器
      <button class="use-hit">  使用按钮透明热区
    </div>
  </div>
</body>
```

- `.stage` 宽度 = `min(100vw, 480px)`，水平居中；高度由 `<img>` 按原始宽高比自动撑开
- 图片比原图高时，页面**纵向滚动**
- `.overlay` 与 `<img>` 是**同一个盒子**（`inset:0`），二者永远重合

## 6. 坐标系与定位技术（关键，必须照做）

所有交互元素相对原图 **941 × 1672** 的像素坐标定位，换算规则固定：

- `.overlay` 设 `container-type: inline-size`，于是 `1cqw = 舞台宽度的 1%`
- **位置**（left/top/width/height）：用 `%` —— `值% = 设计像素 ÷ 对应原图边长 × 100`
- **字号 / 行高 / 与字号挂钩的尺寸**：用 `cqw` —— `值cqw = 设计像素 ÷ 941 × 100`
- 全程**禁止**出现 `px` 常量、`vw`、`background-size:cover`、或拍脑袋的百分比

这样交互层和底图共用一套坐标系、同步缩放，任意屏宽下严丝合缝。

## 7. 三个交互元素规格

### 7.1 剩余次数数字 `.count`

替换原图 `23` 的位置，显示当前剩余次数（初始 `30`，递减至 `0`）。`/30` 属底图、不动。

| 属性 | 值 | 说明 |
|---|---|---|
| 定位 | `right: 46.227%` (=435/941) | 数字**右对齐**，右边缘锚定 x506（紧靠 `/` 左侧） |
| 竖直 | `top: 49.163%` (=822/1672)，`height: 3.110%` (=52/1672) | 字形包络 y822–874 |
| 字号 | `≈ 7.86cqw`（设计 ≈74px） | 目标字形高度 52px；以 `reference/original-design.png` 的 `23` 为准微调 |
| 字体 | 衬线体（serif，偏展示型） | 见 §10 风险项 |
| 颜色 | 线性渐变 上 `#FFF6E2` → 中 `#F2DBAE` → 下 `#E4C088`，`background-clip:text` | 取样金色 #F2DBAE、高光 #FFF6E2 |
| 发光 | 轻微暖色 `text-shadow`，低透明度 | 贴近原图微光，勿过曝 |

文本右对齐，确保 `30 / 9 / 0` 等不同位数时右边缘不动。

### 7.2 使用按钮热区 `.use-hit`

透明可点击层，盖在原图按钮上。**不绘制任何可见按钮**。

| 属性 | 值 |
|---|---|
| `left` | `30.818%` (=290/941) |
| `top` | `58.852%` (=984/1672) |
| `width` | `39.532%` (=372/941) |
| `height` | `5.024%` (=84/1672) |
| 样式 | 完全透明；`cursor:pointer`（可点击时） |

### 7.3 使用记录列表 `.records`

DOM 列表，精确嵌入原图"使用记录"面板的行区域，内部可纵向滚动。

**滚动视口：**

| 属性 | 值 |
|---|---|
| `left` | `21.785%` (=205/941) |
| `top` | `70.694%` (=1182/1672) |
| `width` | `53.879%` (=507/941) |
| `height` | `13.46%`（一页正好 5 行 = 5×45 设计px ÷ 1672） |
| 溢出 | `overflow-y:auto`；一页 5 行，超出可滚动 |
| 滚动条 | 细滚动条（3px、半透明金 `rgba(214,186,130,.55)`）作为可视提示与拖拽手柄；`overscroll-behavior:contain` 防止滚到边界连带翻整页 |

**单行 `.record-row`：**

- 行高：`4.782cqw`（设计 45px）
- 三列（x 相对视口左边缘 x205）：
  - 图标：左起 54–80px（设计），宽约 26px，行内竖直居中
  - 日期：文字左边缘距视口左 109px（设计），格式 `YYYY-MM-DD`
  - 时间：文字右边缘距视口右 31px（设计），右对齐，格式 `HH:MM`
- 文字色 `#C2AAA0`（暖浅灰），字号 `≈ 2.44cqw`（设计 ≈23px）
- 列表**默认为空**，无任何占位文案

## 8. 交互与状态逻辑

### 点击"使用"按钮（仅 `count > 0` 时响应）

1. `count -= 1`
2. 生成新记录：`{ date: 当前真实日期 YYYY-MM-DD, time: 当前真实时间 HH:MM }`
3. 新记录**追加到列表底部**（升序，最新在底）
4. 重渲染数字与列表
5. 列表自动滚动到底部，露出最新一条
6. 写入 localStorage

### `count == 0` 时

按钮热区**不可点击**（`pointer-events:none`，点击无任何响应、无提示、无视觉变化）。

## 9. 初始状态

首次进入（localStorage 无数据）：

- `count = 30`，数字显示 `30`
- 记录列表为空

> 设计稿里的 `23/30` 与那 5 条记录仅为示意，**不是初始值**。

## 10. 数据持久化

- 存于浏览器 `localStorage`，单个 key（如 `script_kill_card_state`）
- 值：`{ count: number, records: [{date, time}, ...] }`
- 进页面先读 localStorage；无则用 §9 初始值
- 刷新后状态保留；不提供重置入口（清除需手动清 localStorage）

## 11. 还原保真度与已知风险

| 部分 | 保真度 |
|---|---|
| 人物 / 背景 / 烫金边框 / 卡片 / 按钮 / 所有标题与静态文字 / `/30` | **100%**（即底图本身） |
| 使用记录文字（日期/时间，普通字体） | 高，可贴近 |
| **剩余次数金色大数字的字体** | **唯一风险点** |

**风险说明：** 金色数字是设计稿里设计过的字形。CSS（衬线体 + 金色渐变 + 发光）可达 95%+；若能拿到设计稿原字体文件，可 100% 一致。除此之外无还原偏差。
